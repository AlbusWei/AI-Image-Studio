## 1. 背景与目标

### 1.1 背景

当前 AI 图片生成领域已有多个成熟的模型 API（Qwen Image 3、GPT-image-2、Nano Banana 2），但缺少一个统一的、专业级的工作站工具来高效调用这些能力。现有工具要么绑定单一模型（如 OpenAI 官方的 ChatGPT 图片生成），要么是面向大众的简化界面（如 Midjourney 的 Discord 交互），无法满足专业生图用户对多模型对比、参数精调、提示词工程、批量生产、以及长期资产管理的深度需求。

用户（产品经理/设计师/内容创作者）在日常工作中面临以下痛点：

1. **多模型割裂**：需要在不同平台间切换来对比 Qwen Image 3、GPT-image-2、Nano Banana 2 的生成效果，prompt 和参数无法复用，效率极低。
2. **提示词工程无沉淀**：好的 prompt 和参数组合散落在各平台的对话历史中，无法系统化管理和复用。每次遇到类似场景需要重新摸索。
3. **批量生产缺失**：现有工具普遍不支持多变体排列组合生成和 prompt 队列批量执行，"跑 50 个 prompt 对比"需要手动逐条操作。
4. **资产管理粗放**：生成的图片缺乏组织（无多级文件夹）、无检索（无语义搜索）、无知识复用（无 RAG 案例库），积累越多越难找到有价值的历史成果。

本产品的目标是构建一个专业级 AI 图片生成工作站，将多模型调用、提示词工程、批量生产、资产管理和知识复用整合在一个统一的工作界面中，形成"生成 → 审视 → 迭代 → 沉淀 → 复用"的完整创作飞轮。

### 1.2 目标用户

| 用户角色 | 特征描述 | 核心诉求 | 使用场景 |
|---------|---------|---------|---------|
| 专业生图用户（唯一用户） | 熟悉 AI 生图工具和 prompt 工程，日常高频使用多个生图模型 API，追求高效创作和知识沉淀 | 多模型统一调用、精确参数控制、批量生产、资产长期管理、提示词知识复用 | 日常创作（单张精调）、批量出图（多变体探索）、案例复盘（知识库积累）、风格探索（跨模型对比） |

### 1.3 业务目标与成功指标

| 目标 | 衡量指标 | 目标值 | 监测方式 |
|------|---------|--------|---------|
| 创作效率提升 | 从"打开工具"到"获得满意的生成结果"的平均时间 | 相比多平台切换工作流降低 50% | 用户自评 + 生成记录时间戳分析 |
| 知识飞轮运转 | 知识库案例包数量月增长率 | ≥20 条/月（稳定使用后） | 知识库入库统计 |
| 模型利用率均衡 | 三个模型的使用分布 | 无明显偏废（各占 20%+） | 生成任务模型分布统计 |
| 资产可检索性 | 在 Gallery 中找到目标图片的平均操作步数 | ≤3 步（搜索/筛选/点击） | 用户操作路径追踪 |

## 2. 需求概述

构建一个专业级 AI 图片生成工作站 Web 应用，支持多模型统一调用（Qwen Image 3 / GPT-image-2 / Nano Banana 2），集成提示词智能扩写、批量生成、异步任务管理、多级文件夹资产管理、知识库 RAG 复用等完整能力，提供分屏工作站式交互体验。

## 3. 产品架构与信息架构

### 3.1 整体布局

Studio 采用**分屏工作站**布局，分为三个核心区域：

- **左侧导航栏**：固定侧边栏，包含导航树（全部图片 / 最近生成 / 收藏 / 文件夹树 / 知识库）和功能入口（生成工作区 / 设置页）。导航栏可折叠为图标模式以扩大工作区面积。
- **中央工作区**：上下文感知的主内容区域。当用户选择"生成"时展示工作区（Workbench）；当选择 Gallery 中的某个文件夹或虚拟视图时展示画廊浏览；当选择知识库时展示案例包管理界面。
- **浮动层**：任务指示器（页面角落）和 Toast 通知，覆盖在所有页面之上，确保后台任务状态全局可见。

## 4. 功能详细设计

### 4.1 生成工作区（Workbench）

**功能描述**：Studio 的核心创作界面，用户在此完成从输入提示词到审视生成结果的全部活动。

**用户故事**：作为专业生图用户，我希望在一个统一的工作区中完成提示词编写、参考图上传、参数配置、模型选择和结果审视，以便高效地进行图片创作和迭代。

**优先级**：P0

#### 4.1.1 工作区布局

工作区由四个子区域组成，采用统一画布 + 模型感知面板的设计：

- **提示词编辑区**：位于工作区上部，包含原始提示词输入框（多行文本）和扩写助手面板。
- **参考图区**：位于提示词编辑区下方或侧方，网格排列的参考图上传槽位。
- **参数面板**：位于参考图区下方或侧方，包含所有可配置的高级参数。
- **结果画板（Result Board）**：位于工作区下部或右侧，展示当前批次的生成结果。

**核心设计原则——统一画布 + 模型感知**：

工作区布局固定不变，切换模型时，不可用的参数控件灰显并显示 tooltip 说明"当前模型不支持此功能"，可用参数自动适配为对应模型的默认值。用户始终能看到完整的参数空间，一眼识别各模型的能力边界。

#### 4.1.2 模型选择与能力矩阵

模型选择器位于工作区顶部（或参数面板顶部），支持快速切换。切换模型时，参数面板根据**模型能力矩阵**自动调整：

| 能力维度 | Qwen Image 3 | GPT-image-2 | Nano Banana 2 |
|---------|-------------|-------------|---------------|
| 生成模式 | T2I（文生图）、I2I（图生图） | T2I、I2I、Inpainting（局部重绘） | T2I、I2I |
| 参考图数量 | 1-3 张 | 1-16 张 | 1 张 |
| 遮罩（Mask） | 不支持 | 支持（PNG + alpha 通道） | 不支持 |
| 尺寸选项 | 由模型自动决定 | 比例（如 16:9、1:1、9:16）或精确像素（16-3840px/边） | 固定 1024×1024 |
| 分辨率档位 | 不适用 | 1K / 2K / 4K（仅比例模式生效） | 不适用 |
| 质量档位 | 不适用 | low / medium / high | 不适用 |
| 单次出图数 | 1 张 | 1-10 张 | 固定 4 张 |
| prompt_extend 参数 | 支持（true/false） | 不支持 | 不支持 |
| Seed 控制 | 取决于模型支持 | 取决于模型支持 | 取决于模型支持 |
| API 模式 | 同步（需本地异步封装） | 异步（原生任务 ID） | 异步（原生任务 ID） |

**交互规则**：

1. 切换模型时，如果当前参考图数量超过目标模型上限，系统提示"当前已有 N 张参考图，[模型名] 最多支持 M 张，请移除多余的参考图"。不移除则无法生成。
2. 切换模型时，已配置的模型特有参数（如质量档位）重置为该模型的默认值，通用参数（如提示词文本）保持不变。
3. 不可用的参数控件视觉灰显 + 鼠标悬浮 tooltip 说明原因。

#### 4.1.3 高级参数面板

参数面板分为"常用参数"（默认展示）和"更多参数"（可折叠）两层：

**常用参数**（默认展示）：

| 参数 | 控件类型 | 说明 |
|------|---------|------|
| 模型选择 | 下拉选择器 | 三个模型 |
| 图片尺寸/比例 | 预设按钮组 + 自定义输入 | 预设：1:1 / 16:9 / 9:16 / 3:4 / 4:3 / auto；自定义：宽×高输入 |
| 生成数量 | 数字步进器 | 根据模型调整范围 |
| 质量（GPT-image-2） | 三档选择器 | low / medium / high |
| prompt_extend（Qwen） | 勾选框 | 透传 API 的自动扩写参数 |

**更多参数**（折叠展开）：

| 参数 | 控件类型 | 说明 |
|------|---------|------|
| Seed | 数字输入 + "随机"按钮 | 控制生成随机性 |
| Mask 局部重绘 | 按钮入口 | 仅 GPT-image-2 可用，点击进入 mask 编辑器 |
| 回调地址 | 文本输入 | 可选，生成完成回调 URL |
| debug 模式 | 勾选框 | Qwen 模型的 debug 参数 |

#### 4.1.4 生成触发

工作区底部（或参数面板底部）放置主操作按钮组：

- **"生成"按钮**（主按钮）：提交当前配置，创建生成任务。按钮状态：可用 / 生成中（显示进度） / 禁用（参数不完整）。
- **"批量生成"按钮**（次按钮 / 下拉展开）：展开批量生成模式选择（多批次 / 多变体 / Prompt 队列），详见 4.4 节。

**业务规则**：

1. 生成前校验：提示词不能为空；参考图数量需在当前模型允许范围内；如有模型特有参数不合法则阻止提交并高亮提示。
2. 提交后，生成任务进入后台任务队列，工作区结果画板显示"生成中"的占位状态（骨架屏或进度条）。
3. 用户可立即切换页面（Gallery、设置等），生成任务继续运行。返回工作区时看到最新结果。
4. 支持同时存在多个生成任务（如：提交一个 T2I 后立即提交一个 I2I）。

**异常处理**：

| 异常场景 | 处理方式 | 用户提示 |
|---------|---------|---------|
| API key 未配置 | 阻止提交，跳转设置页 | "请先在设置中配置 [模型名] 的 API Key" |
| API 返回 429（限流） | 自动进入任务队列重试 | 任务面板显示"限流，自动重试中（第 N 次）" |
| API 返回 402（余额不足） | 任务标记失败 | Toast："API 余额不足，请充值" |
| API 返回 500（服务错误） | 自动重试 1 次，仍失败则标记失败 | Toast："模型服务异常，任务失败" |
| 网络中断 | 自动重试 | 任务面板显示"网络中断，重连后自动重试" |

### 4.2 提示词系统

**功能描述**：管理提示词的输入、扩写、选择和复用，是生成质量的核心杠杆。

**用户故事**：作为专业生图用户，我希望有智能工具帮助我写出更好的提示词，并能复用历史成功案例的 prompt 经验，以便持续提升生成质量。

**优先级**：P0

#### 4.2.1 提示词输入

- 原始提示词输入框为多行文本编辑器，支持长文本（无字数上限，但超过 8000 token 时在底部显示提示"提示词较长，可能影响生成质量"）。
- 输入框右下角显示当前字符数和估算 token 数。
- 支持从剪贴板粘贴、从文件拖入纯文本。

#### 4.2.2 提示词扩写助手（主动扩写 · 主交互）

**入口**：提示词输入框右侧或下方的"扩写"按钮。

**交互流程**：

1. 用户写完原始提示词后，点击"扩写"按钮。
2. 系统调用 LLM（配置的扩写模型，如 Qwen3.7-max），将原始提示词 + RAG 检索到的相似成功案例（来自知识库）作为上下文，生成 2-3 个扩写版本。
3. 扩写结果以卡片列表形式展示在输入框下方或侧面板，每张卡片包含：扩写后的完整 prompt + 一句话说明扩写策略（如"细节丰富版""风格化版""精简聚焦版"）+ 参考的案例包来源。
4. 用户可选择一个版本作为当前提示词（一键替换输入框内容），或在选择后继续手动编辑。
5. 用户可以多次点击"扩写"获取新版本，每次扩写独立，不覆盖上次结果。

**业务规则**：

1. RAG 检索：从知识库中检索与原始提示词语义最相似的 Top-K 案例包（K 可配置，默认 3），作为 LLM 的参考上下文。如果知识库为空，跳过 RAG，使用纯 LLM 扩写。
2. 扩写结果中不包含任何敏感内容或违规内容。
3. 每次扩写记录保留在工作区本地（当前会话内），切换工作区或刷新后清除。

**异常处理**：

| 异常场景 | 处理方式 | 用户提示 |
|---------|---------|---------|
| 扩写 LLM API key 未配置 | "扩写"按钮灰显 | tooltip："请在设置中配置扩写模型的 API Key" |
| LLM API 超时 | 重试 1 次，仍失败则提示 | "扩写服务暂不可用，请稍后重试" |
| 知识库为空 | 正常扩写但跳过 RAG | 卡片上标注"仅基于 LLM 通用能力，未匹配到相似案例" |

#### 4.2.3 自动扩写（prompt_extend 透传 · 辅助开关）

**入口**：参数面板中的勾选框，仅在 Qwen Image 3 模型下可见。

**功能**：勾选后，在发送生图请求时透传 `prompt_extend: true` 参数给 Qwen Image 3 API，由模型端执行自动扩写。

**与主动扩写的关系**：两者可叠加使用。用户可先用主动扩写助手优化 prompt，再勾选 prompt_extend 让模型端做进一步扩展。也可只使用其中一个。工作区中显示当前生效的扩写链路："原始 prompt → [主动扩写] → [prompt_extend] → 最终请求"。

#### 4.2.4 提示词历史

- 工作区保留当前会话的提示词编辑历史（最近 20 条），支持快速回溯切换。
- Gallery 图片详情中显示该图使用的完整提示词（原始 + 扩写），支持"复制 prompt"和"用此 prompt 生成"快捷操作。

### 4.3 参考图管理

**功能描述**：管理用于图生图（I2I）的输入图片，支持上传、预览、排序、角色标注和多来源拖入。

**用户故事**：作为专业生图用户，我希望方便地上传和管理多张参考图，并能从已有资产中快速拖入参考图，以便高效利用视觉参考进行创作。

**优先级**：P0

#### 4.3.1 参考图区域交互

- 参考图区域以紧凑网格展示已添加的参考图缩略图，每张图显示序号。
- 支持的操作：拖拽调整顺序、点击 × 移除、点击放大预览。
- 空状态显示拖放提示区："拖入参考图或点击上传（支持 JPG/PNG/WebP，单张≤50MB）"。
- 当参考图数量较多时（>6 张），网格自动切换为紧凑滚动模式，不挤占工作区空间。

#### 4.3.2 参考图来源

参考图支持三种来源：

1. **本地文件上传**：拖入或点击选择本地文件。
2. **从 Gallery 拖入**：将 Gallery 中的任何图片（生成图或导入的外部图）拖入参考图区域。
3. **从知识库拖入**：将知识库中的案例包图片拖入参考图区域。

#### 4.3.3 角色标注（可选）

- 用户可对任意参考图右键或点击"标注"，设置角色标签。
- 预设角色：风格参考、构图参考、色彩参考、人物参考、场景参考、整体参考。支持自定义角色。
- 角色标注会传入提示词扩写环节，帮助 LLM 理解各参考图的用途（如"图 A 是风格参考，图 B 是构图参考"）。
- 默认不标注也可使用——模型自行综合所有参考图。

**业务规则**：

1. 参考图格式限制：JPG、PNG、WebP。
2. 单张参考图大小限制：≤50MB。
3. 参考图尺寸限制：每边 ≤23170 像素（GPT-image-2 API 限制）。
4. 参考图数量上限随当前选择的模型变化（见模型能力矩阵 4.1.2）。

### 4.4 遮罩与局部重绘（Inpainting）

**功能描述**：在参考图上标记需要重新生成的区域，仅对标记区域进行生成，其余部分保持不变。仅 GPT-image-2 支持。

**用户故事**：作为专业生图用户，我希望只对图片的局部区域进行修改（如修复手部变形、更换背景局部），而不影响已经满意的部分，以便精确控制生成结果。

**优先级**：P0

#### 4.4.1 Mask 编辑器

**入口**：参数面板中的"局部重绘"按钮（仅 GPT-image-2 选中时可点击）。

**交互流程**：

1. 用户点击"局部重绘" → 弹出内建 Mask 编辑器（覆盖在工作区之上或作为模态面板）。
2. 编辑器展示当前第一张参考图，图片上叠加半透明红色蒙版层。
3. 用户使用画笔工具在需要重新生成的区域涂抹（涂抹区域变为红色半透明覆盖）。
4. 编辑器工具栏包含：画笔（添加选区）、橡皮擦（移除选区）、笔刷大小滑块、全选按钮、清除按钮、撤销/重做。
5. 编辑完成后，点击"确认 Mask" → 系统自动生成 PNG mask 图（alpha 通道：涂抹区域透明，未涂抹区域不透明），与参考图关联。
6. 在 prompt 中描述"这个区域应该是什么"，提交生成。

#### 4.4.2 外部 Mask 上传

- 编辑器内提供"上传外部 Mask"入口，用户可上传在其他工具中制作的 mask 图。
- 上传后系统校验：尺寸必须与参考图完全一致；必须包含 alpha 通道；格式为 PNG。
- 校验通过后替换当前编辑器内容，可在此基础上继续编辑。

#### 4.4.3 结果对比

- 局部重绘的生成结果支持与原图的对比视图：左右并排对比或滑块叠加对比。
- 用户可以快速判断重绘区域是否满意，不满意可调整 mask 或 prompt 后重试。

**业务规则**：

1. Mask 尺寸必须与关联的参考图完全一致（宽×高像素匹配），否则 API 拒绝请求。
2. Mask 的透明区域 = 需要重新生成的区域；不透明区域 = 保持不变的区域。
3. 仅 GPT-image-2 支持此功能。选择其他模型时入口灰显 + tooltip："仅 GPT-image-2 支持局部重绘"。

### 4.5 结果展示与批次管理

**功能描述**：展示生成结果，支持批次内审视、对比、标记和操作。

**用户故事**：作为专业生图用户，我希望生成完成后能在工作区内直接审视所有结果，快速标记好图和废图，并对满意的图进行二次操作，以便高效完成创作迭代。

**优先级**：P0

#### 4.5.1 结果画板（Result Board）

- 生成完成后，结果画板以自适应网格展示当前批次的所有图片。4 张 = 2×2 网格，10 张 = 自适应多列网格。
- 生成中显示进度状态：每张图位显示骨架屏/加载动画，完成一张替换一张（GPT-image-2 和 Nano Banana 2 支持渐进式结果返回时）。

#### 4.5.2 图片审视交互

- **点击放大**：点击任何结果图进入大图预览（lightbox 模式），显示完整尺寸。
- **批次内导航**：大图预览中左右箭头切换同批次其他图片。
- **元数据展示**：大图预览的侧栏或底部显示该图的生成参数（prompt、模型、尺寸、质量、seed 等）。

#### 4.5.3 图片快捷操作

大图预览和网格视图中均可触发：

| 操作 | 图标/入口 | 行为 |
|------|----------|------|
| 收藏 | ⭐ 星标按钮 | 切换收藏状态 |
| 淘汰 | 🗑 按钮 | 标记为淘汰（从默认视图中隐藏，不删除文件） |
| 重新生成 | 🔄 按钮 | 用完全相同的参数（含 seed）重新生成 |
| 用作参考图 | 📌 按钮 | 将此图设为参考图，自动跳转到 I2I 模式 |
| 局部重绘 | 画笔按钮 | 仅 GPT-image-2，打开 mask 编辑器 |
| 移动到文件夹 | 📁 按钮 | 选择目标文件夹，移动图片归属 |
| 加入知识库 | 💡 按钮 | 将案例包加入知识库（触发入库流程） |
| 导出/下载 | ⬇ 按钮 | 下载原图 |

#### 4.5.4 批次历史

- 侧边栏导航中增加"生成历史"区域（或在导航栏的"最近生成"视图中），按时间倒序展示所有批次。
- 每个批次显示：封面缩略图（第一张图）+ prompt 摘要（截断显示）+ 模型名 + 生成时间 + 图片数量。
- 点击某批次 → 结果画板切换到该批次的结果展示。

### 4.6 迭代与二次生成

**功能描述**：从已有图片出发进行二次创作，支持多种迭代路径。三层交互设计匹配不同的操作节奏。

**用户故事**：作为专业生图用户，我希望从任何一张已有图片快速发起二次生成（调整 prompt、更换参考图、换模型），以便在"生成 → 审视 → 微调 → 再生成"的创作循环中高效迭代。

**优先级**：P0

#### 4.6.1 快速迭代层（右键/悬浮菜单）

在任何地方看到一张图（结果画板、Gallery、批次历史），右键或悬浮"更多"按钮显示上下文菜单：

| 菜单项 | 行为 |
|--------|------|
| 用相同参数再来一批 | 复制全部参数（含 prompt），更换 seed，提交新的 T2I 任务 |
| 以此图为参考图 | 将此图设为参考图，进入 I2I 模式，prompt 预填为该图历史 prompt |
| 微调 prompt 再生成 | 复制参数到工作区，光标定位到 prompt 输入框供编辑 |
| 局部重绘 | 仅 GPT-image-2，打开 mask 编辑器 |
| 换模型生成 | 子菜单选择目标模型，复制 prompt 并适配参数 |

#### 4.6.2 深度精修层（图片详情面板）

点击任何图片展开详情面板（侧栏或全屏），提供：

- 大图预览 + 完整生成参数。
- 操作按钮栏：修改 prompt 后重新生成、锁定 seed 只改一个参数做对比、以此图为参考图发起 I2I、下载原图、收藏/淘汰/移入文件夹/加入知识库。
- 用户备注输入框：可为图片添加文字备注。

#### 4.6.3 拖拽参考图层（工作区交互）

- 当工作区已打开时，用户可从 Gallery、批次历史或知识库中拖拽图片到工作区的参考图槽位。
- 拖入后自动切换到 I2I 模式（如果当前是 T2I 模式），prompt 预填为该拖入图片的历史 prompt（如有）。
- 多个参考图拖入时，按拖入顺序排列。

**关键区分**：

- "用相同参数重新生成"= T2I（相同 prompt + 相同参数，不同 seed，得到同方向但不同细节的图）。
- "以此图为参考图"= I2I（将图片的视觉内容作为输入，配合新 prompt 做风格/内容变化）。

两者意图和结果不同，UI 上必须用不同的图标和文案明确区分。

### 4.7 批量生成

**功能描述**：支持三种批量生成模式，覆盖从"扩大候选池"到"系统性参数探索"到"离线批量执行"的完整场景。

**用户故事**：作为专业生图用户，我希望批量提交大量生成任务并自动管理调度（限流、重试、断点续跑），以便大规模探索创意方向或批量生产素材。

**优先级**：P0（多批次）/ P1（多变体）/ P1（Prompt 队列）

#### 4.7.1 模式一：多批次（P0）

**场景**：同一 prompt + 参数，重复生成多批以扩大候选池。

**交互**：

1. 在生成按钮旁的"批量生成"下拉中选择"多批次"。
2. 输入重复次数（如"5 批"）。
3. 系统创建 N 个独立的生成任务，依次提交到任务队列。
4. 结果按批次组织：批次 1、批次 2、…、批次 N，每批独立展示。

#### 4.7.2 模式二：多变体（P1）

**场景**：一个基础 prompt + 一组变量，系统自动排列组合生成。

**交互**：

1. 选择"多变体"模式。
2. 定义变量集：
   - **Prompt 变量**：在 prompt 中使用 `{变量名}` 占位符，定义变量值列表（如 `{风格}` = ["胶片风", "水彩风", "赛博朋克"]）。
   - **参数变量**：选择要变化的参数及其值列表（如尺寸 = [1:1, 16:9, 9:16]）。
3. 系统计算排列组合总数并展示"将生成 N 张图（X 组合 × 模型单次出图数）"。
4. 用户确认后提交批量任务。
5. 结果以矩阵视图展示：行 = prompt 变体，列 = 参数变体，单元格 = 生成结果。

#### 4.7.3 模式三：Prompt 队列（P1）

**场景**：多个不同 prompt 排队依次生成。

**交互**：

1. 选择"Prompt 队列"模式。
2. 以列表形式输入多个 prompt（每行一个，或从文件导入 CSV/TXT）。
3. 可为每个 prompt 独立设置参数，也可统一参数。
4. 系统显示队列总览："N 个 prompt 排队，预计 M 分钟完成"。
5. 提交后进入任务队列。

#### 4.7.4 任务调度与容错

三种批量模式共享统一的调度层：

- **并发控制**：根据模型 API 的限流规则控制并发请求数，避免触发 429。
- **自动重试**：429 限流自动退避重试；5xx 错误自动重试 1 次。
- **断点续跑**：如果批量任务中途中断（网络断开、应用重启），恢复后从中断点继续，已完成的不重复执行。
- **进度可见**：批量任务在任务面板中显示总进度（"完成 8/20，排队中 5，失败重试 1"）。
- **暂停/取消**：用户可随时暂停或取消正在执行的批量任务。已完成的批次不受影响。

### 4.8 后台任务管理

**功能描述**：全局管理所有生成任务的生命周期，确保后台任务在页面切换时不中断，并提供进度可见性和完成通知。

**用户故事**：作为专业生图用户，我希望提交生成任务后能自由浏览其他页面而不中断任务，并在任务完成时收到通知，以便高效利用等待时间。

**优先级**：P0

#### 4.8.1 任务后台化（核心约束）

**产品约束**：不管底层模型 API 是同步还是异步，在用户感知层面，所有生成都遵循"提交 → 后台运行 → 完成通知"的模式。用户切换页面（Gallery、设置、知识库）时，进行中的生成任务继续运行，结果可回看。

#### 4.8.2 浮动任务指示器

- 位于页面右下角（或其他不干扰主内容的角落位置）。
- 显示当前进行中的任务数量（数字角标）。
- 悬浮时显示一行摘要（如"2 个任务进行中，1 个排队"）。
- 无任务时指示器自动隐藏。

#### 4.8.3 任务面板

点击任务指示器展开任务面板（侧滑面板，不遮挡主内容），展示四个分区的任务：

**进行中**：
- 每个任务显示：prompt 摘要 + 模型名 + 进度条 + 预估剩余时间（如可获取）。
- 操作：暂停、取消。

**排队中**：
- 等待被调度的任务列表。
- 操作：调整优先级（上移/下移）、移除。

**已完成**：
- 最近完成的任务（默认显示最近 10 个）。
- 点击任意已完成任务 → 跳转到该批次的结果画板。

**失败**：
- 显示失败原因（限流/余额不足/服务错误/网络中断）。
- 操作：重试、移除。

#### 4.8.4 完成通知

- **Toast 通知**：任务完成时在页面右下角弹出轻量通知，3 秒自动消失。包含 prompt 摘要 + "查看"按钮。点击"查看"跳转结果画板。
- **浏览器通知**：如果用户授权了浏览器通知权限，即使不在 Studio 页面也能收到完成通知。
- **声音提示**（可选）：设置页可开关，默认关闭。

### 4.9 Gallery 画廊模块

**功能描述**：Studio 的内容浏览与管理中心，承载所有已生成图片、导入的外部参考图的组织、检索和浏览。

**用户故事**：作为专业生图用户，我希望在一个功能强大的画廊中浏览、搜索和组织我积累的所有图片资产，以便快速找到需要的图片并高效管理。

**优先级**：P0

#### 4.9.1 默认视图：时间线 + 智能分组

- Gallery 默认以时间线视图展示，按时间倒序排列。
- **智能分组**：系统将同一次创作操作产生的所有批次自动归为一个"批次会话（Batch Session）"。每个会话折叠显示为：封面缩略图 + prompt 摘要 + 图片数量 + 时间。
- 点击展开会话 → 显示该会话内所有批次的图片。
- 时间线粒度可切换：按日 / 按周 / 按月。

#### 4.9.2 搜索

顶部搜索栏支持：

| 搜索类型 | 说明 | 示例 |
|---------|------|------|
| 关键词搜索 | 在 prompt 文本、用户标注中匹配关键词 | "胶片风人像" |
| 语义搜索 | 用自然语言描述意图，系统理解语义而非仅匹配关键词 | "暖色调的户外女性写真，背景有花和阳光" |
| 以图搜图 | 上传一张图或从图库选一张，查找视觉上相似的图片 | 拖入一张参考图 → 返回相似风格的生成记录 |

搜索类型通过搜索栏左侧的图标切换（默认关键词，可切换到语义/以图搜图）。

#### 4.9.3 筛选

侧边栏或搜索栏下方提供筛选条件：

- 模型（Qwen Image 3 / GPT-image-2 / Nano Banana 2）
- 日期范围（今天 / 最近 7 天 / 最近 30 天 / 自定义）
- 尺寸比例（横版 / 竖版 / 方形 / 自定义）
- 分辨率档位（1K / 2K / 4K）
- 收藏状态（仅收藏 / 全部）
- 文件夹（当前选中文件夹的子范围）

筛选条件可叠加。搜索结果在筛选范围内执行。

#### 4.9.4 图片操作

Gallery 中的图片支持：

- 单击 → 进入图片详情面板。
- 右键 → 上下文菜单（同 4.6.1 的快速迭代层 + 移动/收藏/淘汰/导出）。
- 多选（Ctrl/Cmd + 点击 或 框选） → 批量操作栏出现：批量移动到文件夹、批量收藏、批量导出、批量淘汰。
- 拖拽 → 拖入工作区参考图槽位，或拖入文件夹树进行归类。

#### 4.9.5 图片导入

Gallery 支持导入外部图片作为参考素材：

- 拖入本地文件或点击"导入"按钮上传。
- 导入的图片进入 Gallery 的"导入素材"默认文件夹（可在设置中修改）。
- 导入图片仅保留图片文件本身，无生成元数据。

### 4.10 文件夹与组织系统

**功能描述**：多级目录结构管理图片的物理归属，配合收藏和虚拟视图提供多维度的组织方式。

**用户故事**：作为专业生图用户，我希望用多级文件夹来组织我的图片资产，并通过收藏和虚拟视图快速访问重要内容，以便在大量积累后仍保持清晰的资产结构。

**优先级**：P0

#### 4.10.1 文件夹管理

- 左侧导航栏的文件夹树支持无限级嵌套。
- 操作：新建文件夹、重命名、删除（文件夹内图片移至"未分类"）、拖拽调整层级。
- 每张图片有且仅有一个所属文件夹。默认归属"未分类"文件夹。
- 移动图片到新文件夹：拖拽到文件夹树、右键菜单选择、或在图片详情面板中选择。

#### 4.10.2 虚拟视图

虚拟视图不改变图片物理归属，基于条件动态聚合展示：

| 虚拟视图 | 聚合条件 | 说明 |
|---------|---------|------|
| 全部图片 | 无筛选 | Gallery 全量展示 |
| 最近生成 | 最近 7 天内生成 | 快速访问近期成果 |
| 收藏 | 收藏标记 = true | 全局精选内容 |

用户不可自定义虚拟视图（与文件夹区分——文件夹是物理组织，虚拟视图是系统预设的快捷入口）。

#### 4.10.3 收藏

- 任何图片可一键收藏/取消收藏（星标按钮）。
- 收藏状态独立于文件夹——图片可以在任何文件夹中被收藏，收藏后在"收藏"虚拟视图中集中展示。
- 收藏是进入知识库的前置条件——只有被收藏的图片才能被加入知识库。

### 4.11 知识库与 RAG

**功能描述**：从收藏中精选高质量案例组成知识库，作为提示词扩写 RAG 检索的素材源，形成"生成 → 沉淀 → 复用"的创作飞轮。

**用户故事**：作为专业生图用户，我希望把生成效果最好的案例系统性地积累下来，并在未来创作类似场景时自动获得参考建议，以便持续提升生成质量。

**优先级**：P1

#### 4.11.1 知识库管理界面

导航栏"知识库"入口进入，展示所有案例包：

- 网格视图：案例包封面 + prompt 摘要 + 模型名 + 入库时间。
- 列表视图：紧凑行显示，适合快速浏览大量案例。
- 搜索和筛选：按 prompt 关键词搜索、按模型筛选、按入库时间筛选。

#### 4.11.2 入库流程

**前置条件**：图片已被收藏。

**入库触发**：

1. 在图片详情面板中点击"加入知识库"按钮。
2. 在 Gallery 中右键 → "加入知识库"。
3. 在"收藏"虚拟视图中批量选中 → 批量入库。

**入库时的自动打包**：

系统自动采集该图片的完整生成上下文，组装为案例包：

| 案例包字段 | 来源 | 是否可编辑 |
|-----------|------|-----------|
| 生成图片 | 自动 | 否 |
| 原始提示词 | 自动（生成时记录） | 是 |
| 扩写提示词 | 自动（如有扩写记录） | 是 |
| 参考图列表 | 自动 | 否 |
| 模型名 | 自动 | 否 |
| 全部参数 | 自动 | 否 |
| 用户标注 | 自动生成草稿 + 用户编辑 | 是 |

**用户标注的辅助生成**：

入库时，系统自动调用多模态 LLM（如 Qwen3.7-plus），输入为：生成图片 + 原始 prompt + 参数。输出一段结构化标注草稿，包含：场景描述、画面特征分析、风格关键词、可复用的 prompt 技巧。

用户可在此基础上修改确认。标注辅助使用的提示词模板作为系统配置项维护（可在设置页查看和编辑）。

**入库确认**：

展示案例包预览（含自动打包的字段 + 标注草稿），用户确认后入库。也可选择跳过预览直接一键入库。

#### 4.11.3 案例包编辑

入库后可随时编辑：

- 用户标注文本。
- 原始提示词和扩写提示词（如发现更优的表达方式可以修正，修正后的版本同时用于 RAG 检索）。

不可编辑的字段：生成图片本身、模型名、参数（这些是历史记录事实）。

#### 4.11.4 RAG 检索与反馈循环

- 知识库中的案例包自动建立索引，用于提示词扩写助手的 RAG 检索（详见 4.2.2）。
- 检索维度：提示词文本语义相似度（主维度）+ 模型匹配度（辅维度）。
- 案例包数量越多、覆盖场景越广，RAG 扩写质量越高——形成飞轮。

#### 4.11.5 移除知识库

- 从知识库移除案例包不会删除图片或取消收藏，仅从 RAG 检索范围中移除。
- 移除操作在案例包详情或列表视图中执行。

### 4.12 存储与同步

**功能描述**：管理图片的本地和云端存储，采用冷热分层策略平衡访问速度和存储成本。

**用户故事**：作为专业生图用户，我希望近期图片能快速本地访问，历史图片自动迁移到云端以节省本地空间，同时保证所有图片都有云端备份不会丢失。

**优先级**：P0

#### 4.12.1 冷热分层策略

| 层级 | 本地存储 | 云端存储 | 访问体验 |
|------|---------|---------|---------|
| 热区 | 原图 + 缩略图 + 元数据 | 原图 + 缩略图 + 元数据 | 打开即看，零延迟 |
| 冷区 | 仅缩略图 + 元数据 | 原图 + 缩略图 + 元数据 | Gallery 浏览无影响（缩略图），点进去查看原图需 1-2 秒从云端加载 |

**热区容量管理**：

- 默认值基于可用磁盘空间智能推荐（如可用空间的 10%），用户可手动配置。
- 典型配置：100GB（约容纳 10,000-20,000 张高清图）。
- 系统根据实际生成速率动态提示："热区已用 78%，预计 X 周后开始迁移旧图到冷区"。

**自动迁移**：

- 当热区容量达到上限时，系统按时间顺序（最早的先迁移）将原图从本地移至云端，保留本地缩略图和元数据。
- 迁移过程在后台执行，不影响用户操作。
- 迁移进度在设置页可见。

**知识库和收藏的特殊处理**：

- 知识库中的案例包图片始终保留本地原图（不迁移到冷区），确保 RAG 检索和快速访问。
- 收藏图片同理，始终保留本地原图。
- 这意味着热区容量计算中需预留知识库 + 收藏的空间。

#### 4.12.2 云端配置

- 在设置页配置云端存储：阿里云 OSS Bucket 名称、Region、AccessKey。
- 首次配置时引导创建 Bucket（提供推荐配置：标准存储、私有读写）。
- 连接状态实时显示在设置页和 Gallery 状态栏。

#### 4.12.3 冷图访问

- 用户在 Gallery 中点击一张冷区图片时：
  1. 立即显示缩略图放大版（模糊预览）。
  2. 后台从云端拉取原图。
  3. 原图加载完成后替换为清晰版本（1-2 秒）。
  4. 可选：拉取后缓存到本地临时目录（设置页可配置缓存大小），下次访问无需重新下载。

#### 4.12.4 云端灾备

- 所有生成的图片在生成完成后自动上传到云端（热区图片上传原图，冷区图片已在云端）。
- 上传失败自动重试。上传状态在图片详情中可见（"已同步到云端" / "同步中" / "同步失败"）。

### 4.13 导入与导出

**功能描述**：支持图片、元数据、知识库、文件夹结构和配置的完整导入导出，确保数据不被锁死在工具中。

**用户故事**：作为专业生图用户，我希望能够完整地导出我的所有数据（图片、prompt、知识库、组织结构），以便在需要时迁移到其他工具或设备。

**优先级**：P1

#### 4.13.1 导出

**图片导出**：

- 单张下载：图片详情面板的下载按钮，下载原图。
- 批量下载：在 Gallery 中多选后点击"导出"，按文件夹结构打包为 zip 下载。可选是否包含缩略图。

**元数据导出**：

- 导出为 JSON 文件，每张图片一个对象，包含：prompt（原始+扩写）、模型名、全部参数、生成时间、文件夹路径、收藏状态、知识库标注（如有）。
- 可独立于图片导出（只导出元数据，不含图片文件）。

**知识库导出**：

- 导出知识库所有案例包为 JSON 包，包含完整的案例包结构（图片引用或内嵌 base64、prompt、参数、标注）。
- 支持选择性导出（勾选部分案例包）。

**文件夹结构导出**：

- 包含在元数据导出中（每张图片的 `folder_path` 字段）。

**配置导出**：

- 导出设置页的所有偏好配置为 JSON 文件（不含 API key 等敏感信息）。

**完整导出（一键迁移）**：

- 一键打包所有数据：图片（zip）+ 元数据（JSON）+ 知识库（JSON）+ 配置（JSON），生成一个完整的迁移包。

#### 4.13.2 导入

- 支持导入上述任何导出格式。
- 知识库导入：将 JSON 包中的案例包合并到当前知识库（冲突处理：prompt 相同的案例包视为重复，提示用户选择覆盖或跳过）。
- 文件夹结构导入：根据 `folder_path` 自动创建对应的文件夹树。
- 配置导入：恢复偏好设置（需手动重新填入 API key）。

### 4.14 设置与配置

**功能描述**：集中管理所有外部服务连接和应用偏好。

**用户故事**：作为专业生图用户，我希望在一个清晰的设置页面中管理所有模型 API 的凭据、云端存储连接和应用偏好，以便快速配置和维护。

**优先级**：P0

#### 4.14.1 首次引导（Setup Wizard）

首次打开 Studio 时自动触发引导流程：

1. **欢迎页**：简要介绍 Studio 功能。
2. **模型配置**：选择要使用的模型（可多选） → 逐个填入 API Key 和 Endpoint → 点击"测试连接"验证 → 成功/失败反馈。
3. **存储配置**（可跳过）：配置阿里云 OSS → 测试连接。
4. **扩写 LLM 配置**（可跳过）：配置扩写模型的 API Key。
5. **基本偏好**：热区大小、默认模型。
6. 完成 → 进入工作区。

引导完成后，用户随时可在设置页中修改所有配置。

#### 4.14.2 设置页结构

设置页以 Tab 组织：

**Tab 1：模型 API**

每个模型一个卡片，包含：
- 模型名称 + 状态指示（已连接/未配置/连接失败）。
- API Endpoint 输入框。
- API Key 输入框（默认掩码显示，点击"眼睛"图标切换明文/掩码）。
- "测试连接"按钮 → 发起一个最小化的测试请求，显示结果。
- 模型默认参数（默认质量、默认尺寸等）。

**Tab 2：存储**

- 本地存储路径配置。
- 热区容量配置：滑块 + 数字输入 + "智能推荐"按钮（基于当前磁盘空间计算推荐值）。
- 热区使用状态：已用 / 总量 + 可视化进度条。
- 阿里云 OSS 配置：Bucket 名称、Region、AccessKey ID、AccessKey Secret（眼睛切换）。
- 连接状态 + "测试连接"按钮。
- 冷图缓存配置：缓存大小上限。

**Tab 3：提示词扩写**

- 扩写 LLM 模型选择（下拉列表）。
- API Endpoint + API Key。
- RAG 参数：Top-K 检索数量（默认 3）。
- 标注辅助的提示词模板编辑器（多行文本框，显示当前模板，可编辑）。

**Tab 4：通用**

- 默认生成模型。
- 图片默认保存格式。
- 浏览器通知权限开关。
- 声音提示开关。
- 界面语言（中文/English）。
- 深色/浅色模式。

### 4.15 快捷键体系

**功能描述**：为高频操作提供全局和上下文相关的键盘快捷键，覆盖生成工作区、Gallery 浏览、图片审视、Mask 编辑等核心场景，显著提升专业用户的操作效率。

**用户故事**：作为高频使用的专业生图用户，我希望通过键盘快捷键完成大部分常用操作而不必频繁移动鼠标寻找按钮，以便保持创作心流和操作节奏。

**优先级**：P1

#### 4.15.1 设计原则

1. **冲突规避**：不与浏览器原生快捷键冲突（如 Ctrl+W 关闭标签页、Ctrl+T 新建标签页、Ctrl+R 刷新页面等），也不拦截操作系统级快捷键。
2. **可发现性**：所有快捷键在对应的菜单项、按钮 tooltip 中标注。设置页提供"快捷键速查表"页面，显示全部快捷键及其功能描述。首次使用时可通过引导提示展示核心快捷键（如 `?` 键随时打开快捷键速查浮层）。
3. **上下文感知**：快捷键按当前焦点区域生效。同一个按键在不同上下文（工作区 vs Gallery vs Lightbox vs Mask 编辑器）有不同含义，不会产生歧义。
4. **可自定义**（P2）：设置页提供快捷键重映射界面，用户可将任何快捷键绑定为自己习惯的按键组合。冲突检测：如果新绑定与已有快捷键或浏览器原生快捷键冲突，给出警告。

#### 4.15.2 全局快捷键

在任何页面（工作区、Gallery、知识库、设置）均生效：

| 快捷键 | 功能 | 说明 |

| `Ctrl/Cmd + Enter` | 提交生成 | 无论当前焦点在哪，触发工作区的生成操作 |
| `Ctrl/Cmd + Shift + G` | 进入生成工作区 | 从任何页面快速切换到 Workbench |
| `Ctrl/Cmd + Shift + F` | 聚焦搜索栏 | 从任何页面快速激活 Gallery 搜索 |
| `Ctrl/Cmd + Shift + T` | 打开任务面板 | 展开浮动任务面板查看任务状态 |
| `?` | 快捷键速查 | 打开/关闭快捷键速查浮层（输入框有焦点时不触发） |
| `Esc` | 关闭/退出 | 关闭当前打开的浮层（Lightbox / Mask 编辑器 / 任务面板 / 快捷键速查 / 设置页）；如无浮层则无操作 |
| `1` / `2` / `3` | 快速切换模型 | 仅在工作区且非输入状态时生效：1=Qwen Image 3, 2=GPT-image-2, 3=Nano Banana 2 |

#### 4.15.7 快捷键速查浮层

- 按 `?` 键从任何页面打开。
- 浮层按场景分组展示所有快捷键（全局 / 工作区 / Gallery / Lightbox / Mask 编辑器）。
- 当前所在场景的快捷键组高亮显示。
- 点击浮层中的某条快捷键可跳转到对应功能的详细说明。
- 按 `Esc` 或 `?` 关闭浮层。

**业务规则**：

1. 文本输入框（prompt 输入框、搜索栏、设置页输入框等）获得焦点时，单键快捷键（`1/2/3/F/R/E/B/I/M/?` 等）不触发，避免干扰文字输入。组合键（`Ctrl/Cmd + *`）正常触发。
2. Mask 编辑器激活时，快捷键优先级最高（覆盖 Gallery 和 Lightbox 的快捷键含义）。
3. 快捷键冲突处理：如果同一按键在不同上下文有不同含义，以当前焦点区域为准。焦点判定优先级：Mask 编辑器 > Lightbox > 工作区输入框 > Gallery 网格 > 全局。

## 8. 模型 API 参考（附录）

本附录记录各模型 API 的关键差异，供开发实现参考。PRD 产品设计部分不涉及这些技术细节。

### 8.1 Qwen Image 3

- **Endpoint**：`POST https://poc-dashscope.aliyuncs.com/api/v1/services/aigc/multimodal-generation/generation`
- **认证**：Bearer Token（Authorization Header）
- **T2I 请求**：content 中仅包含 `{"text": "prompt"}`
- **I2I 请求**：content 中包含 1-3 个 `{"image": "url"}` + 1 个 `{"text": "prompt"}`
- **特殊参数**：`prompt_extend`（boolean，模型端自动扩写）、`debug`（boolean）
- **API 模式**：同步（需本地封装为异步任务）

### 8.2 GPT-image-2

- **Endpoint**：`POST /v1/images/generations`（EvoLink API）
- **认证**：Bearer Token
- **请求参数**：model、prompt（≤32000字符）、image_urls（1-16张）、mask_url（PNG+alpha）、size（比例或像素）、resolution（1K/2K/4K）、quality（low/medium/high）、n（1-10）、callback_url
- **图片限制**：单张 ≤50MB，每边 ≤23170px，格式 JPEG/PNG/WebP
- **像素预算**：655,360 ≤ 宽×高 ≤ 8,294,400
- **API 模式**：异步（返回 task ID + status + progress）
- **结果有效期**：24 小时（需及时下载持久化）

### 8.3 Nano Banana 2

- **Endpoint**：`POST /v1/images/generations`（EvoLink API）
- **认证**：Bearer Token
- **请求参数**：model、prompt（≤32000字符）、image_url（1张，≤50MB）、size（固定 1024×1024）
- **固定输出**：每次 4 张图
- **API 模式**：异步（返回 task ID + status + progress）
- **结果有效期**：24 小时

## 11. 产品决策记录（附录）

本附录记录产品设计过程中的关键决策及其理由，对应 grill session 的 17 轮讨论。

### PDR-001：Solo 使用定位

**决策**：产品定位为单人使用的专业工作站，不包含多用户认证、权限管理、团队协作等功能。

**理由**：用户是个人专业生图用户，使用自己的 API 账号和阿里云资源。Solo 定位可跳过认证/权限复杂度，让所有 UX 决策聚焦在单人创作效率上。

### PDR-002：分屏工作站布局

**决策**：采用左侧导航 + 中央工作区的分屏工作站布局，而非 Tab 页面切换或单 Feed 流。

**理由**：专业创作工具（Lightroom、Capture One）都将"工作表面"和"资产库"保持同屏可见，因为创作循环需要在工作（生成）和审视（Gallery）之间快速切换。Tab 切换 (b) 在迭代循环中造成上下文断裂；Feed 流 (c) 在大量图片后失去组织性。

### PDR-003：统一画布 + 模型感知面板

**决策**：工作区布局固定，切换模型时不可用参数灰显而非隐藏，保持完整参数空间可见。

**理由**：用户需要快速跨模型对比同一 prompt 的效果，统一布局减少切换时的认知成本。灰显而非隐藏让用户始终知道各模型的能力边界。

### PDR-004：生成任务后台化

**决策**：所有生成在用户感知层面遵循"提交→后台运行→完成通知"模式，页面切换不中断任务。

**理由**：生图耗时较长（数十秒到数分钟），用户不应被迫在等待页面上空等。后台化释放等待时间用于浏览 Gallery、整理资产或提交新任务。

### PDR-005：提示词扩写混合模式

**决策**：以主动扩写助手 (b) 为主交互，Qwen API 的 prompt_extend 参数透传为辅助开关，对话式澄清 (c) 不在 Studio 内建。

**理由**：专业用户需要精确控制 prompt 内容，纯自动扩写 (a) 不可见不可编辑不利于调试。主动扩写让用户看到、选择、编辑扩写结果。对话式需求澄清用 QoderWork 技能外部完成更合适。

### PDR-006：文件夹 + 收藏 + 知识库三层组织

**决策**：文件夹为物理归属（一图一位），收藏为全局标记，知识库为收藏的精选子集。

**理由**：文件夹提供确定性组织（不需要记忆标签），收藏提供跨文件夹快捷访问，知识库提供 RAG 质量门槛（不是所有好图都值得作为参考案例）。三层漏斗越往上质量越高。

### PDR-007：冷热分层存储

**决策**：热区保留本地原图（约 100GB 可配置），冷区仅保留云端原图 + 本地缩略图。知识库和收藏始终保留本地原图。

**理由**：90% 的浏览集中在近期图片，热区覆盖这些图片的零延迟访问。冷图从云端拉取 1-2 秒可接受。知识库和收藏是高频精选资产，必须快速访问。

### PDR-008：Mask 编辑器 + 外部上传

**决策**：内建轻量 Mask 编辑器（画笔/橡皮/撤销），同时支持外部 mask 图上传。使用开源画布编辑器方案。

**理由**：内建编辑器覆盖 80% 的局部修改需求，避免离开 Studio 中断工作流。外部上传满足精细需求。开源方案成熟可靠，无需自研。

### PDR-009：集中设置页

**决策**：所有配置集中在独立的设置页管理，不在各功能界面嵌入配置项。

**理由**：配置调整是低频操作，不值得在各处散布配置入口。集中页提供全局概览（哪些模型已配通、存储空间状态），也便于首次使用的引导流程。

### PDR-010：全量数据可迁移

**决策**：支持图片、元数据、知识库、文件夹结构、配置的完整导出导入，不做分享链接功能。

**理由**：数据自由是专业工具的底线保障。Solo 用户不需要分享链接，但需要完整的迁移能力以防工具锁定。

### PDR-011：上下文感知快捷键体系

**决策**：建立五层上下文感知快捷键体系（全局 / 工作区 / Gallery / Lightbox / Mask 编辑器），按焦点区域优先级生效，文本输入状态自动屏蔽单键快捷键，提供 `?` 速查浮层和 P2 阶段的可自定义能力。

**理由**：专业工具的高频操作（生成、切换模型、收藏、浏览）需要键盘驱动以保持操作节奏和心流。上下文感知避免按键冲突和误触，焦点优先级确保嵌套场景（如 Lightbox 中打开 Mask 编辑器）的行为可预测。速查浮层降低学习成本。可自定义（P2）尊重个人习惯。

## 12. 开放问题

以下问题在产品设计阶段识别但尚未最终决策，需在后续阶段明确：

1. **缩略图策略**：缩略图的生成时机（生成完成后立即生成？首次浏览时懒生成？）、尺寸规格、存储位置（与原图同目录？独立缩略图目录？）。
2. **图片淘汰机制**：被标记"淘汰"的图片在物理层面如何处理？保留文件但从 UI 隐藏？还是移入一个"已淘汰"特殊文件夹供反悔？
3. **知识库容量上限**：知识库理论上无上限，但 RAG 检索性能在案例包数量极大时可能下降。是否需要设定软上限或引入分层检索策略？
4. **多变体批量的变量上限**：排列组合可能产生大量任务（如 5 风格 × 3 光线 × 4 构图 = 60 组合）。是否需要设定单次批量上限？
5. **prompt 模板**：是否支持用户保存常用 prompt 模板（如"胶片风人像模板""产品摄影模板"），生成时一键调用？这不同于知识库案例包——模板是 prompt 骨架，案例包是完整案例。
6. **多语言提示词**：三个模型对中英文 prompt 的理解能力可能不同。是否需要在扩写助手中加入"自动翻译"能力（中文 prompt → 英文版本）？
