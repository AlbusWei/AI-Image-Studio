应用采用双进程架构：
- Main 进程 (`electron/main.cjs`)：负责初始化 SQLite（`sql.js`）、启动内嵌 HTTP API 代理服务器（`api-server.cjs`，监听 `127.0.0.1:0` 随机端口并注册 `/api/qwen|evolink|oss|llm|proxy-image` 路由）、注册自定义 `app://` 协议、创建 BrowserWindow、管理 OSS 增量同步（`oss-sync.cjs`）以及窗口生命周期。
- Renderer 进程（React SPA）：入口 `src/main.jsx` 异步初始化数据库后端后挂载 `<App />`；`App.jsx` 通过 `react-router-dom` HashRouter 路由到 `pages/*`，外层包裹全局能力组件（TaskPanel/Lightbox/MaskEditor/ShortcutOverlay）。

数据层使用策略模式抽象（`src/db/database.js`）：按 `window.electronAPI?.db` 是否存在自动选择后端——Electron 下走 `electron-backend.js`（经 `preload.cjs` 暴露的 IPC 调用 main 进程的 SQLite），浏览器下走 `dexie-backend.js`（IndexedDB/Dexie）。`preload.cjs` 将 db/fs/oss/app 三类能力以 `window.electronAPI.*` 命名空间安全暴露给渲染进程，严格遵循 `nodeIntegration: false` + `contextIsolation: true` 的安全模型。

依赖方向：`pages → components/stores/services → db/constants`，store 不直接依赖 UI，services 不依赖 zustand，保持单向数据流。构建产物由 `electron-builder.yml` 打包为 Windows 安装包。
