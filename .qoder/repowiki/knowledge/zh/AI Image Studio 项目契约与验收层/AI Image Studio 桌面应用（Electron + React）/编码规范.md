- Zustand store 使用 `create((set, get) => ({...}))` 定义，修改 state 一律通过 `set(produce(state => {...}))` 不可变更新，并在 setter 末尾调用 `get().saveSettings()` 自动落盘。
- API 适配层采用工厂模式：`services/api/index.js` 的 `getModelAdapter(modelId)` 根据 modelId 返回对应适配器实例，新增模型只需添加 case 分支。
- 页面级组件通过 `React.lazy(() => import('./pages/xxx'))` 按需加载，并由顶层 `<Suspense fallback={<LoadingSkeleton />} />` 统一兜底渲染骨架屏。
- 全局 UI 能力（任务面板、灯箱、遮罩编辑器、快捷键提示）在 `AppInner` 中统一挂载，通过 `useUIStore` 暴露的布尔开关控制显隐，避免在各页面重复实现。
- 样式变量集中来自 `components/ui/spark-tokens.css`，组件内仅使用 CSS 变量（如 `var(--accent-primary)`、`var(--sp-4)`）而非硬编码色值或间距。
- Electron 安全边界：所有主进程能力通过 `contextBridge.exposeInMainWorld('electronAPI', {...})` 以命名空间形式暴露，IPC 通道名采用 `db:*` / `fs:*` / `oss:*` / `app:*` 前缀分组，渲染进程不得直接访问 `ipcRenderer`。
