双进程架构：
- Main 进程（`electron/main.cjs`）按固定顺序初始化：注册特权 `app://` 协议 → 启动 SQLite（`sql.js`，数据目录 `app.getPath('userData')`）→ 注册 IPC 处理器（`ipc-handlers.cjs`/`file-manager.cjs`）→ 启动内嵌 HTTP API 代理服务器（`api-server.cjs`，开发固定端口 19527、生产随机端口并通过 `app:getApiPort` 暴露给渲染端）→ 初始化 OSS 增量同步（`oss-sync.cjs`，监听 `powerMonitor.resume` 自动重试）→ 创建 BrowserWindow 并在首次 `did-finish-load` 触发 IndexedDB → SQLite 迁移（`migration.cjs`）。
- Renderer 进程（React SPA）入口 `src/main.jsx` 先异步 `initDatabase()` 选择后端再挂载 `<App />`；`App.jsx` 通过 `react-router-dom` HashRouter 路由到 `pages/*`，外层包裹全局能力组件（TaskPanel/Lightbox/MaskEditor/ShortcutOverlay）。
- 数据层策略模式（`src/db/database.js`）：优先 Electron IPC（`window.electronAPI.db`），其次 HTTP 后端（`http-backend.js` 调 main 进程 api-server），最后 Dexie/IndexedDB 兜底；所有导出函数转发到当前 backend，调用方零感知。
- 安全边界：`preload.cjs` 仅通过 `contextBridge.exposeInMainWorld('electronAPI', {...})` 暴露 db/fs/oss/app 四类命名空间，IPC 通道名以 `db:*` / `fs:*` / `oss:*` / `app:*` 前缀分组，渲染端禁止直接访问 `ipcRenderer`。
- 依赖方向：`pages → components/stores/services → db/constants`，store 不依赖 UI，services 不依赖 zustand，保持单向数据流。构建产物由 `electron-builder.yml` 打包为 Windows 安装包。
